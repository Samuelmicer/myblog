    TileFlow: A Framework for Modeling Fusion Dataflow via Tree-based Analysis

    • 作者：郑思泽，梁云
    • 单位：北京大学
    • MICRO'23

    Timeloop是一个数据流性能模型，通过写加速器架构配置，软件计算问题描述，以及软硬件的映射文件（mapping），Timeloop就可以给出一个估计的执行性能。这种性能模型，对于AI加速器设计，数据流映射设计，都是有积极意义的。相关的工作还有MAESTRO (MICRO'20)，TENET (ISCA'21)等等。不过这些数据流性能模型都只关注一个算子的映射问题。一个算子，其实就是一个完美循环，是相对容易数学建模和分析的问题。但是现在我们面对的数据流很少有单个算子的了，大部分情况下都要考虑多个算子融合起来。从现在的AI芯片设计也可以看到，NV GPU有Cuda Core和Tensor Core，一边做MMA计算，一边做Vector运算；NPU也是Matrix Core和Vector Core一起使用。这样，我们做一个MMA+Epilogue的融合就非常自然。近年来由于Transformer架构成为DNN主流，很多融合技术都为self-attention结构和MLP结构做了特殊设计，比如FlashAttention，我之前的HPCA工作Chimera也是这个范畴。所以我们认为，现在的数据流性能分析框架也应该演进了，不能再只考虑一个算子。演进的积极意义，一方面是可以针对将来出现的新的架构，设计数据流的时候，可以提供一些性能模型指导，反过来，对于某一融合数据流，设计专门的硬件架构的时候，也可以用这样的模型去估计一下资源怎么分配能满足性能和功耗需求。TileFlow就是基于这样的想法做出来的框架，我们的分析方法是基于Timeloop的方法的演进和扩展。(作者注)

    Chimera: An Analytical Optimizing Framework for Effective Compute-intensive Operators Fusion

    • 作者：郑思泽，梁云
    • 单位：北京大学
    • HPCA'23

    长久以来，我们关注自动程序优化的三个方面：auto-schedule, auto-tensorize, auto-fuse。FlexTensor的工作认为异构系统上的张量程序优化是可以自动化的，在常规的auto-tune基础上，可以进一步自动组合优化原语从而将用户从优化任务中彻底解放；AMOS工作则是讲auto-tensorize问题的自动化进行解决，通过提出指令抽象、匹配算法、定制化代码生成方案实现用户无需关心如何使用特殊指令就可以得到这些特殊硬件的加速福利；Chimera是这系列工作的最后一块拼图，将优化问题拓展到多个算子的场景，将fusion工作的自动化实现出来。总结来说，Chimera是做算子融合优化的，用户提供的是原生的TVM Tensor Expression，得到的结果是一个自动优化而来的融合kernel。Chimera内部容纳了FlexTensor和AMOS的技术积累，同时具有auto-schedule和auto-tensoriz的部分能力。(作者注)

    FlexTensor: An Automatic Schedule Exploration and Optimization Framework for Tensor Computation on Heterogeneous System

    • 作者：郑思泽，梁云
    • 单位：北京大学
    • ASPLOS'20

    本文介绍了名为FlexTensor的用于异构系统上张量计算调度探索和优化的框架。利用启发式算法和机器学习方法，系统地对不同硬件、不同调度组成的 设计空间进行探索优化，寻找最优的调度配置。测试了12种不同的张量计算，在CPU、GPU、FPGA等平台上进行了测试。
